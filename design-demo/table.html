TABLE OUTLINE — 3 PROPOSALS

colors + font unchanged everywhere · same cell sizes as production · pick one (or mix elements) · toggle top-right for light mode

CURRENT (reference)

1px rounded frame, faint hairlines on every cell edge, 2px header rule, 4px dark bands between regions. Functional but busy — the grid competes with the color coding.

A · FLOATING CHIPS

No borders anywhere. Cells become rounded chips separated by 3px of page background — the swell colors do all the structure (GitHub-heatmap feel). Region groups separate by a slightly larger gap. Softest, most modern read; the NOW outline pops harder against pure background than against hairlines.

B · DAY-BANDED MINIMAL

Vertical hairlines removed entirely — hours flow together and a single stronger rule marks each midnight, so the eye navigates by day instead of by hour. Horizontal hairlines kept but fainter. Table floats inside a padded card (10px radius) instead of touching the frame. Calmest option; best for scanning swell events that span many hours.

B2 · DAY-BANDED, SQUARED

Same idea as B minus the bubble: no padding gutter (table meets the frame), radius 10px → 4px, single 1px header rule. Keeps what matters — no vertical hairlines, hours fuse into color runs, one 1.5px rule at each midnight, faint horizontal hairlines, dark region-separator bands as in the current table. Crisper and closer to the existing look.

C · REGION CARDS

Each buoy-region row becomes its own rounded card (1px ring, 7px radius) with clear space between cards — the strongest row rhythm, matching how you actually read the table (one region at a time). Vertical hairlines stay inside each card; header floats free above.
